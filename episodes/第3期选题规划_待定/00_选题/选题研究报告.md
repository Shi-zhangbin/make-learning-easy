## 三、内容结构规划

### 布局阶段

#### P0 开场（1min）
- 回顾第2期3个类比：🛋️宜家套装（镜像）→🍳独立厨房（容器）→🎮USB-C（--device）
- "上次我们看懂了图纸，今天把螺丝刀拿起来干活"
- 展示"今天要做的事"——一张全流程地图

#### P1 概念速览：一张地图看清全部（2-3min）
把完整流程画在一张图上：

```
你的电脑 (SSH) 
  → 远程服务器 (华为云)
    → Docker拉镜像 (docker pull xxx)
      → Docker跑容器 (docker run --device...)
        → 容器里启动模型推理服务
          → 调优：调整参数看性能变化
```

**关键概念让观众记住：**
- Docker = 一键部署工具（不再是宜家套装，是"一个按钮让软件运行"）
- 容器 = 一个隔离的小房间
- --device挂载NPU = 给这个小房间插上AI加速卡
- 模型推理 = 给模型输入，它算输出
- 调优 = 调整配置让算得更快/省内存

#### P2 动手第一步：连服务器+检查环境（2min）
```
# 主人已经会的操作
ssh user@server
npu-smi info    # 检查NPU状态
```

**讲解**：
- npu-smi就像GPUs里的nvidia-smi
- 看Chip Count、Memory Usage
- 确认环境正常

#### P3 动手第二步：拉取镜像（3min）
```
docker pull {待定的镜像地址}
```

**讲解**：
- 不展开技术细节（第2期已讲），只说"这是下载准备好的软件包"
- 看下载过程：分层下载，显示进度
- docker images 查看已下载的镜像

#### P4 动手第三步：启动容器（4min）
```
docker run -it --rm \
  --device /dev/davinci0 \
  --device /dev/davinci1 \
  -p 8000:8000 \
  {镜像名称} \
  bash
```

**讲解**：
- --rm: 用完就删，不占空间（类比：酒店退房）
- --device: 把NPU设备传给容器（第2期讲的USB-C连接）
- -p: 端口映射（类比：给房间开个门，外面能进来访问）
- 进入容器后的shell提示符变化 → 确实是"另一个世界"

#### P5 动手第四步：启动模型推理服务（3min）
```
# 在容器内
python -m vllm.entrypoints.openai.api_server \
  --model {模型路径} \
  --trust-remote-code \
  --max-model-len 8192
```

**讲解**：
- vLLM是推理框架，帮我们把模型高效跑起来
- 等日志出现"Uvicorn running on http://0.0.0.0:8000" → 就绪
- 讲解：这一步是把模型"加载到NPU内存中"

#### P6 动手第五步：测试推理+看性能（3min）
```
# 测试请求（另一个终端）
curl http://localhost:8000/v1/chat/completions \
  -H "Content-Type: application/json" \
  -d '{
    "model": "{模型名}",
    "messages": [{"role": "user", "content": "你好，你是谁？"}],
    "max_tokens": 100
  }'
```

**看性能指标**（调优前）：
- 首token延迟（Time to First Token）
- 吞吐量（每秒输出token数）
- 显存占用

#### P7 动手第六步：调优（3min）
```
# 换个参数再跑
python -m vllm.entrypoints.openai.api_server \
  --model {模型路径} \
  --trust-remote-code \
  --max-model-len 8192 \
  --max-num-batched-tokens 16384 \
  --max-num-seqs 8
```

**直观对比**：
| 参数 | 改前 | 改后 | 效果 |
|------|------|------|------|
| max-num-seqs | 4 | 8 | 并发能力↑ |
| max-model-len | 8192 | 4096 | 显存占用↓ |
| gpu-memory-utilization | 默认0.9 | 0.95 | 更充分利用NPU |

**展示**：调参前后的吞吐量对比，让观众看到数值变化

#### P8 收尾总结（1min）
- 今天干了什么：SSH → docker pull → docker run → 跑推理 → 调优
- 6条核心命令回顾
- 下期预告：多卡部署 / 量化 / 微调

## 六、风险提醒

1. **模型下载太慢** → 提前准备，建议预先下载好或使用lite版本
2. **vLLM升腾版兼容性** → 提前确认vllm-ascend版本是否支持该模型
3. **命令太长看不清** → 分屏展示（左边命令+右边效果）
4. **调优效果不明显** → 选差异明显的参数，不要微调效果
5. **观众掉队** → 每步操作后停顿，说"你现在看到的输出应该长这样"
6. **第2期观众不连贯** → 开场用15秒快速回顾三个类比
